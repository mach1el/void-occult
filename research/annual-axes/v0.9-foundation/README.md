# Annual Axes V0.9 Foundation

Unified research foundation for a possible future Nam Phái Annual Axes V0.9:
a full audit of the current production **V0.8** engine, plus a source-graded
doctrine/policy pack for what a V0.9 candidate might change.

**This is not a V0.9 implementation.** No production formula, weight, clamp,
registry, fixture, or UI file was modified to produce this pack. See
`V0.9-FOUNDATION-DECISION.md` for the final readiness verdict.

## Layout

```
audit/       generated (deterministic) audit artifacts + hand-authored findings
sources/     source registry, claim registry, review/contradiction logs
policy/      7 policy matrices (school, domain-palace, star-domain,
             temporal-identity, dignity, thai-tue, unsupported-star)
schema/      JSON Schemas for the 4 core registries/contracts above
prompts/     handoff prompts for the next research/implementation phase
```

## Regenerating the audit artifacts

```bash
npm run research:annual-axes-v09:audit-fast    # smoke test, always runs
npm run research:annual-axes-v09:audit-full    # full 100x12 corpus, writes audit/*.json
npm run research:annual-axes-v09:validate      # schema + cross-reference checks
npm run research:annual-axes-v09:rule-coverage # rule/capability coverage summary
npm run research:annual-axes-v09:report        # gate + mass + readiness summary
```

`audit-full` is deterministic — running it twice produces byte-identical
JSON (verified; see `src/lib/ziwei/analysis/modules/annual-axes/audit/v0.9/__tests__/v09-determinism.test.ts`).

## Where the real numbers came from

Every number in `audit/*.json` was generated by actually running the
production V0.8 engine (`analyzeAnnualAxes`) over the existing deterministic
100-chart × 12-year corpus contract
(`src/lib/ziwei/analysis/modules/annual-axes/audit/build-audit-corpus.ts` →
`FULL_CORPUS_CONTRACT`), not estimated or hand-written. `audit/audit-findings.md`
walks through the most important findings with exact figures and links each
one to a research question, per the task's linkage requirement.

## Sourcing status (read before trusting any doctrinal claim)

This pack has **zero** verified classical/school-manual/published-reference
sources. Every doctrine-adjacent claim in `sources/claim-registry.v0.9.json`
is `engineering-hypothesis`, `derived` (from this audit's own measurements),
`disputed`, or `unsupported` — never `classical`. See
`sources/source-review-log.md` for why, and
`prompts/v0.9-candidate-handoff-prompt.md` for what would need to happen to
change that.
